# TenderApp

A production SaaS design system for a public procurement platform. TenderApp transforms complex tender data into clear, actionable workflows for procurement professionals.

The design system's signature is **"frosted professional"** — glass-morphism over a fixed SVG background pattern creates depth and lightness, while a data-dense typography scale (12-20px) and the Untitled UI color palette keep the interface trustworthy and precise. Built on shadcn/ui + Radix UI primitives with Tailwind CSS v4.

This is a real-world production design system, not a fictional example. It demonstrates how DESIGN.md can document an existing, actively developed application — including custom sections ("Glass & Frosted Effects"), component variant tables, and an extensive "Do's and Don'ts" enforcement section for AI coding agents.

## Files

| File | Description |
|------|-------------|
| `DESIGN.md` | The complete design system specification in DESIGN.md format, including both structured YAML design tokens (frontmatter) and human-readable style guidance (markdown body). Includes custom sections for glass-morphism effects and agent-specific enforcement rules. |
| `tailwind.config.js` | A Tailwind CSS v3 theme configuration derived from the design tokens in the DESIGN.md frontmatter. Covers colors, typography, border-radius, and spacing. |
| `design_tokens.json` | A [Design Tokens Community Group](https://www.designtokens.org/) JSON file containing all design tokens from the DESIGN.md frontmatter. |
